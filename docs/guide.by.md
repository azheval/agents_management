# Кіраванне агентамі

## База дадзеных

Для запуску кантэйнера з базай дадзеных выкарыстоўваць каманду

```make
make db-up
```

або

```cmd
docker-compose up -d
```

Пры першым запуску неабходна стварыць табліцы базы дадзеных, выканаўшы міграцыю

```make
make db-migrate-fresh
```

або

```cmd
docker exec -i agent_db psql -U admin -d agent_management < db/migrations/001_full_schema.sql
```

## Старт сервера і агента

Для старту сервера выкарыстоўваць каманду

```cmd
<шлях>/server.exe -config <шлях>/config.json
```

Прыклад файла `config.json` у `server/config.json`.

Для старту агента выкарыстоўваць каманду

```cmd
<шлях>/agent.exe
```

Пры старце агента ствараецца сістэмны файл `agent.id` з унікальным ідэнтыфікатарам агента.

### Стварэнне задачы

Новая задача ствараецца з формы `Tasks` камандай `Create new task`.
У форме стварэння задачы выбіраюцца агенты, якія будуць выконваць задачу. Для кожнага выбранага агента будзе створана свая ўнікальная задача.
Паказваецца тып задачы. Для кожнага тыпу задачы патрабуецца запоўніць унікальныя палі.

#### EXEC_COMMAND

![img_002.png](/docs/img/img_002.png)

Запаўняецца апісанне задачы, каманда (напрыклад, `C:/ws/tech_log/go/techlog-stat/.dist/techlog-stat.exe`), аргументы каманды паказваюцца праз коску (напрыклад, `sdbl-context,--input,C:/ws/tech_log/logs,--glob,*/*.*,--output,C:/ws/tech_log/out/sdbl_agent,--top,10`).

#### EXEC_PYTHON_SCRIPT

![img_003.png](/docs/img/img_003.png)

Запаўняецца апісанне задачы, выбіраюцца неабходныя файлы (не толькі python файлы), паказваецца файл аркестрата (Entrypoint), які будзе выкліканы агентам на выкананне.
Усе паказаныя файлы пры стварэнні задачы будуць змешчаны ў архіў, перададзены агенту, разархіваваны і выкананы.
**Важна**. Python павінен быць папярэдне ўсталяваны на аддаленым працоўным месцы.

#### FETCH_FILE

![img_004.png](/docs/img/img_004.png)

Запаўняецца апісанне задачы, шлях да файла для агента, шлях, па якім сервер захавае файл.

#### PUSH_FILE

![img_005.png](/docs/img/img_005.png)

Запаўняецца апісанне задачы, шлях захавання файла агентам, файлы, якія перадаюцца серверам.

#### AGENT_UPDATE

![img_006.png](/docs/img/img_006.png)

Запаўняецца апісанне задачы, паказваецца файл новай версіі агента. Кантрольная сума разлічваецца аўтаматычна.

У любой задачы можна наладзіць блок `Alert Pipeline` які адпраўляе струмень `stdout` у тэлеграм.

![img_010.png](/docs/img/img_010.png)

![img_011.png](/docs/img/img_011.png)

Выбіраецца неабходны графік выканання каманды, паказваецца таймаўт на выкананне каманды:

- Immediate. Задача выконваецца неадкладна.
- Scheduled. Задача будзе выканана ў паказаны дзень і час.
- Recurring. Задача будзе выконвацца перыядычна па зададзеным графіку.
- Chained. Задача будзе выканана пасля выканання папярэдняй (выбранай) задачы.
